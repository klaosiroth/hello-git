Hello Git

This is the first file in my new Git Repo.

      git --version

Configure Git

Note: Use global to set the username and e-mail for every repository on your computer. If you want to set the username/e-mail for just the current repo, you can remove global

      git config --global user.email "you@example.com"
      git config --global user.name "Your Name"
    

      git config --list

Initialize Git

Initialize Git on that folder:

      git --init

Git Adding New Files

README.md file that describes the repository (recommended for all repositories):

List the files in our current working directory:

      ls

Then we check the Git status and see if it is a part of our repo:

      git status
      git status --short

Note: Short status flags are:

  • ?? - Untracked files
  • A - Files added to stage
  • M - Modified files
  • D - Deleted files

Git Add

      git add <file_name>
      git add .
      git add --all (or) git add -A
      git add *<files_type>

Git Reset

Undo a git add

      git reset <file_name>

Git Commit

Since we have finished our work, we are ready move from stage to commit for our repo.
When we commit, we should always include a message. 

      git commit -m "message"

Git Commit Log

To view the history of commits for a repository, you can use the log command:

      git log

Git Help

There are a couple of different ways you can use the help command in command line:

  • git command -help - See all the available options for the specific command
  • git help --all - See all possible commands
      git command -help
      git help --all
